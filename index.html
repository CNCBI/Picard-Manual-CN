一个用来处理高通量测序（HTS）的数据和格式的Java命令行工具箱。

Picard是通过使用HTSJDK Java 库 HTSJDK来实现的，支持用来存储高通量测序的数据的常见的文件格式，比如 SAM 和 VCF

Introduction - 简介

SAM（序列比对/Map）格式是一个用来存储长核苷酸序列比对的一种格式。在 hts-specs 页面里面描述了SAM和与它相关的文件格式。

Picard由下列两样构成：基于Java的来处理SAM文件的命令行工具，和一个Java API (HTSJDK) （Java应用程序接口），这个API可以用于创建可以读写SAM文件的新软件。它支持SAM文本格式和SAM二进制格式 (BAM)。

To get help using and developing Picard, please use the GATK Forums.

Picard is hosted by Github.com. The source code is available at the Picard project page here.

You can check out the latest source code with:

git clone git@github.com:broadinstitute/picard.git

The source code for HTSJDK is available on GitHub here. You can check out the latest source code with: 

git clone git@github.com:samtools/htsjdk.git

Enjoy!

Quick Start

See Building and Overview for details.

export $JAVA_HOME=Java_6_or_7_install_path
ant -lib lib/ant package-commands
java jvm-args -jar dist/picard.jar PicardCommandName OPTION1=value1 OPTION2=value2...

Getting Help

Picard is supported through the GATK Forums. Join the GATK Forum to get support for Picard programs among other tools supported by the Broad Institute.

Before Asking For Help

Before posting to the Forum, please do the following:

  • Try the latest version of Picard
  • See if your problem is discussed in the Frequently Asked Questions
  • Search the GATK Forums to see if your problem was previously discussed.
  • Run PicardValidateSamFile MODE=SUMMARY. Attempt to resolve or at least understand any problems reported.

When sending email about a problem, please include the following:

  • Command line(s) you used.
  • Program console output and metrics files. Repetitive console output may be abbreviated.
  • Entire stack trace if one was produced.
  • Version of JVM are you using (obtained by running 'java -version').
