一个用来处理高通量测序（HTS）的数据和格式的Java命令行工具箱。

Picard是通过使用HTSJDK Java 库 HTSJDK来实现的，支持用来存储高通量测序的数据的常见的文件格式，比如 SAM 和 VCF

Introduction - 简介

SAM（序列比对/Map）格式是一个用来存储长核苷酸序列比对的一种格式。在 hts-specs 页面里面描述了SAM和与它相关的文件格式。

Picard由下列两样构成：基于Java的来处理SAM文件的命令行工具，和一个Java API (HTSJDK) （Java应用程序接口），这个API可以用于创建可以读写SAM文件的新软件。它支持SAM文本格式和SAM二进制格式 (BAM)。

如需获取Picard的使用或开发方面的帮助，请使用 GATK 论坛.

Picard托管在 Github.com上。 源代码可以在Picard项目页面下载。

你可以使用下面的命令检出最新的源码：

git clone git@github.com:broadinstitute/picard.git

HTSJDK的源码可以在GitHub上面下载到。 你可以使用下面的命令检出最新的源码： 

git clone git@github.com:samtools/htsjdk.git

尽情享用！

Quick Start - 快速开始

更多细节，请查看 Building 和 概述。

export $JAVA_HOME=Java_6_or_7_install_path
ant -lib lib/ant package-commands
java jvm-args -jar dist/picard.jar PicardCommandName OPTION1=value1 OPTION2=value2...

Getting Help - 获取帮助

Picard通过 GATK 论坛 提供支持。 请加入GATK论坛，以获取Picard软件和其他由Broad Institute支持的软件的帮助信息。

Before Asking For Help - 在寻求帮助之前

在论坛上面提问之前，请做下列事情：

  • 尝试最新版本的Picard。
  • 看看你的问题是否在常问问题中讨论过了。
  • 在 GATK 论坛 中搜索，看看你的问题是否已经在之前讨论过了。
  • 运行Picard ValidateSamFile MODE=SUMMARY。尝试解决或至少理解报告的任何问题。

在发邮件问一个问题时，请包含以下信息：

  • 您使用的命令。
  • 程序控制台的输出和 metrics 文件。可以缩减重复的输出。
  • 完整的堆栈跟踪信息 (stack trace) ，如果生成了的话。
  • 您正在使用的 JVM 的版本（通过运行 'java -version' 来获取）。
